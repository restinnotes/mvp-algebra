## 2024-04-08 - Unmemoized Render Bottlenecks in PracticeUI
**Learning:** `PracticeUI.tsx` suffered from unnecessary O(N) operations on every re-render (such as fast keystrokes in the search bar). This included iterating over full district and exam type lists to calculate unique arrays (`Set` creations and string mappings) directly in the render function, as well as an unused `weakKPs` variable calculating mastery metrics on every keystroke. Additionally, the complex string-filtering logic for rendering the displayed questions was evaluated inline instead of being memoized based on the actual search query and data changes.
**Action:** Always wrap heavy data derivations—especially `.map()`, `Set()` creations, `.sort()`, and expensive `.filter()` arrays derived from state or props—in `useMemo()` inside high-frequency UI components to prevent blocking the main thread during rapid user input. Dead code representing expensive operations should be immediately stripped to save cycles.
