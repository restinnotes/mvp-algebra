# 产品功能规格书：AI 数学认知外骨骼 (中考压轴专题)

## 产品定位
聚焦中考数学 **Q18（填空压轴）、Q24 & Q25（函数/几何压轴）**。通过**思路导向的碎裂化交互**，精准捕获并修复腰部学生在这些"必丢分点"上的认知 Bug。

---

## 学生端核心体验流程 (MVP 阶段)

### Phase 1：进入与拆解
1. 学生输入一道真题（Q18, 24, 25）。
2. 系统后台执行**影子解题 (Shadow Solve)**，生成带逻辑锚点的路径。
3. 将题目的骨架**碎裂化**，映射到原子知识点。

### Phase 2：思路首发 (Socratic Entrance)
4. **必须先问思路**：系统禁止直接进入计算，先询问学生的解题方案。
5. **思路对齐**：比对学生思路与标准路径。若缺失关键逻辑（如：未意识到 Q18 需分类讨论），AI 进行苏格拉底式追问。

### Phase 3：碎裂化逐步执行
6. 学生在表单中填入各步骤结论。
7. **逻辑校验**：大模型执行对错判定。
8. **支架介入**：若犯错，触发针对该步骤细解的引导支架。连续错 3 次自动降级熔断。

### Phase 4：定性复盘与归因
9. 做完整题后，系统执行 **Review**。
10. **核心归因**：归纳错误的定性特征（符号习惯、逻辑盲区、条件缺失），更新长期记忆（LTM）。

### Phase 5：考核通关式出题
11. 针对薄弱点，从**往年真题库**中拉取考核相同细则知识点的题目（或由 AI 补写高质量同类题）。
12. 必须在**零提示**下独立完成，方可通关。

---

## 实施原则

*   **UI 优先级**：低。保持基础功能可用即可，核心资源投入在逻辑链路。
*   **出题策略**：优先使用真实往年卷素材，确保考核细则的专业性。
*   **模型底座**：以 **Gemini 3.0 Flash** 作为基准测试模型，确保 Prompt 引导逻辑的鲁棒性。
